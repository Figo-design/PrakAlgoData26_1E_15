Jawaban
1. Tidak, class yang akan dibuat array of object tidak harus selalu memiliki atribut dan sekaligus method. Karena array of object hanya menyimpan referensi object, cukup memiliki struktur class yang bisa dibuat object, meskipun hanya berisi atribut saja.
2. Mendeklarasikan array of object dan membuat array yang menyimpan 3 object mahasiswa.
3. Class mahasiswa tetap memiliki konstruktor yaitu default konstruktor yang dibuat otomatis oleh java, hal tersebut menyebabkan new mahasiswa tetap bisa dipanggil meskipun kita tidak menulisnya.
4. Menginstansiasi obejct mahasiswa yang disimpan dalam array dan mengisi nilai atribut object tersebut.
5. Class mahasiswa dan mahasiswa demo dipisahkan karena memiliki tanggung jawab masing-masing. Class mahasiswa berfungsi sebagai blueprint(model data) sedangkan class mahasiswa demo berfungsi sebagai eksekutor(menjalankan program). Selain hal tersebut, ada tujuan lain yaitu agar kode lebih rapi dan tersruktur, bisa digunakan kembali, serta mudah untuk dikembangkan.
